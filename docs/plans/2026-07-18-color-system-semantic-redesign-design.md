# 色彩系統語意重構 — 設計文件

日期：2026-07-18

## 背景

深色主題目前給人「偏藍綠」的觀感。追查後發現這是兩個獨立來源疊加，而在盤點過程中又挖出一個比觀感更嚴重的正確性問題。

**觀感來源一 — 底色飽和度過高。** 深色表面全部取自 Tailwind Slate：背景 `#0F172A` 為 HSL(222°, 47%, 11%)、卡片 `#1E293B` 為 HSL(217°, 33%, 17%)。222° 與 217° 是純藍而非藍綠，但 47% 的飽和度對一個承擔「中性背景」職責的顏色而言過高，因此讀起來是藍色而不是深灰。Slate 是 Tailwind 五組中性色階（Slate / Gray / Zinc / Neutral / Stone）中最偏冷的一組。

**觀感來源二 — 強調色落在青綠。** `secondaryColor = #03DAC6` 為 HSL(174°, 97%, 43%)，是 Material Design 2 深色主題規範的預設 secondary，並非針對本專案的設計決策。它出現在 section header 漸層、更新進度橫幅、設定頁漸層、空狀態圖示、技術指標選擇器等 20 處。

**正確性問題 — 綠色語意超載。** 本專案採台股慣例（紅＝漲、綠＝跌），但綠色目前同時承擔六種意義：

| 色值 | 語意 | 色相 |
|:---|:---|---:|
| `#2ED573` | 下跌 | 145° |
| `#4CAF50` | 成功／高信心 | 122° |
| `#27AE60` | 股利正面 | 145° |
| `#10B981` | 低波動／OBV | 160° |
| `#8BC34A` | 籌碼偏多 | 88° |
| `#4ADE80` | 更新成功 | 142° |

其中 `#27AE60`（股利正面）與 `#2ED573`（下跌）色相完全相同，僅明度不同。

**最嚴重的一項 — 籌碼評等色階方向相反。** `chip_strength_indicator.dart` 將籌碼強勢對應綠色、籌碼弱勢對應紅色，採用的是國際慣例（綠＝好），與本專案其餘部分的台股慣例相反。個股詳情頁會同時渲染帶紅綠股價的 header（`stock_detail_screen.dart`）與綠色「強勢」徽章（`chip_tab.dart`），使用者在籌碼分頁上，畫面上緣的紅色代表「上漲、好事」，下緣的紅色代表「籌碼弱勢、壞事」。

## 設計原則

顏色依**語意類別**而非用途分組，類別決定可用色相範圍。

| 類別 | 判準 | 可用色相 |
|:---|:---|:---|
| **Directional** | 這個值變大代表偏多還是偏空？ | 紅 355° ／ 綠 145° ／ 灰（平盤） |
| **Quality** | 有好壞之分但無多空方向 | 紫 255° ／ 灰 |
| **Category** | 純分類，無好壞也無方向 | 紅區綠區以外任意色相，序列間需保持最小間距（見下） |
| **Warning** | 「請注意」，與多空無關 | 琥珀 38-46° |

紅綠是稀缺資源，只有 Directional 類別可以使用。這條原則同時解釋了為何籌碼評等應該翻轉（籌碼強弱本來就是多空方向，與漲跌屬同一語意軸），以及為何「高信心」「更新成功」應該離開綠色（它們不是多空訊號）。

## 色盤

所有對比度為 WCAG 2.1 相對亮度精算值，非估算。背景 `#18181B`、卡片 `#27272A`。

### 表面階層 — Tailwind Zinc

| 用途 | 色值 | 來源 |
|:---|:---|:---|
| 背景 | `#18181B` | Zinc 900 |
| 卡片／表面 | `#27272A` | Zinc 800 |
| Chip／分隔線 | `#3F3F46` | Zinc 700 |
| 次要文字 | `#A1A1AA` | Zinc 400 |
| 主要文字 | `#F4F4F5` | Zinc 100 |

飽和度自 47% 降至 4%，背景不再與股價綠色競爭色相。

### 品牌 — Violet

深色主題採 Material 3 色調邏輯：primary 使用淺色調，onPrimary 使用深色。原方案以 `#8B5CF6` 作填色配白字，實測僅 4.23:1，不符 AA 標準，已修正。

| 角色 | 色值 | 對比 | 用途 |
|:---|:---|---:|:---|
| `primary` | `#A78BFA` | 6.51:1 | 按鈕底、選中態、文字、圖示 |
| `onPrimary` | `#18181B` | 6.51:1 | 紫底上的文字 |
| 裝飾 | `#8B5CF6` | — | 邊框、≤0.2 alpha 底色、focus ring |

`#8B5CF6` 對背景僅 4.18:1，不得承載文字。

### Directional

|:---|:---|---:|---:|

| 平盤 | `#A1A1AA` | 6.91:1 | 5.81:1 |

### Quality

| 用途 | 色值 |
|:---|:---|
| 高信心／更新成功／股利正面／釘選論點有效 | `#A78BFA` |
| 低波動／低強度／停用 | `#71717A` |

### Warning

| 用途 | 色值 | 色相 | 對背景 |
|:---|:---|---:|---:|
| 警示 | `#F59E0B` | 38° | 8.25:1 |
| 注意 | `#FCD34D` | 46° | 12.29:1 |

移除法人三色後，UI 語意層的暖色只剩這兩個。圖表調色盤仍可能取用暖色相（可用空間不足以避開），但圖表序列色只出現在圖表內部，不與警示徽章同框。

### 法人 — 移除分類色

外資／投信／自營商的專屬顏色移除，圖示與邊框統一使用 `#A1A1AA`（對卡片 5.81:1）。

理由：這三個顏色實際只用於 14px 圖示、8px 圓點與 alpha 0.3 邊框，且每個實例都緊鄰文字標籤；趨勢圖只繪製一條線（外資＋投信合計），不存在三序列疊圖需要區分的情況。顏色在此是冗餘的第三重編碼，移除不損失資訊。買賣超金額本來就依正負套用方向色，這部分維持不變。

此決定同時消除四組色相過近的問題：

| 組合 | 間距 | 說明 |
|:---|---:|:---|
| 自營橘 27° vs 上漲紅 355° | 32.2° | **同一張卡片**上橘色圖示配紅色數字 |
| 自營橘 27° vs 警示琥珀 38° | 10.7° | |
| 自營橘 27° vs 注意黃 46° | 18.9° | |
| 投信洋紅 292° vs 品牌紫 255° | 33.9° | |

## 命名結構

顏色常數按語意類別重新分組命名，使呼叫端在取用時即可判斷自己拿到的是哪一類：

```dart
PriceColors.up / .down / .flat / .chipRating(rating)   // 唯一可用紅綠者
QualityColors.high / .dividend / .pinnedActive / .low
CategoryColors.chart(index)
WarningColors.warning / .caution
```

命名本身不具強制力，強制力來自下節的守門測試。選擇重新命名而非完整型別分層，是因為 Flutter 的 `Color` 難以真正封死（呼叫端隨時可寫 `Colors.green`），完整分層需改動 100 處以上呼叫端，投入產出比不足。

## 測試策略

三層守門測試，全部 TDD 紅先行：

1. **色相守門** — 斷言 `QualityColors`、`CategoryColors`、`WarningColors` 的每個色值，色相不落在紅區（>345° 或 <15°）與綠區（88°-175°）。這是防止未來漂移的主防線。

2. **對比度守門** — 斷言所有前景色對其配對背景的對比 ≥4.5:1（正常文字）或 ≥3.0:1（≥18px 粗體）。品牌紫 `#8B5CF6` 的 4.18:1 就是被這層測出來的。

3. **語意完備性** — 斷言 `ChipRating` 五個 enum 值各自對應 Directional 色，且順序單調（強勢端在紅、弱勢端在綠），防止只改一半造成色階中斷。

顏色常數改值不涉及邏輯斷言，現有測試不受影響。

## 已知張力

**圖表調色盤餘裕最少。** 排除紅區（30°）與綠區（87°）後，可用色相僅剩 243°。`DesignTokens.chartPalette` 目前有 8 色，平均間距 30°，恰好落在「過近」判定門檻。`price_overlay_chart` 情況最嚴重——那是股價疊圖，以綠色繪製第四檔股票的線會被讀成「該檔在跌」。

處理方式：縮減為 6 色。6 色分佈於 243° 的平均間距為 40.5°，故序列間最小間距訂為 **≥35°**——這是空間允許的上限，無法比照三色情境要求 ≥60°。序列超過 6 個時改用直接標註而非增加顏色。這是整套色盤中把握度最低、最可能在實作階段需要調整的部分。

`industry_allocation_card` 另有一份 13 色以上的產業色表（含 `#22C55E` 金融保險業、`#14B8A6` 航運業），同樣需依色相規則重排，但產業配置圓餅圖不涉及股價語境，優先級低於 `price_overlay_chart`。

**上漲紅對比 4.46:1，差 0.04 未達 AA。** 此為既有色值而非本次引入。股價數字一律為 ≥15px 粗體，適用大字門檻 3.0:1，實際有充足餘裕。調整台股紅色屬於另一層級的決定，本次不處理。

## 不做的事

- 不調整 `upColor` / `downColor` 的色值。台股慣例的紅綠是使用者既有認知，不因對比度小數點差異而更動。
- 不採用完整型別分層封裝顏色。投入產出比不足，且無法阻止呼叫端直接使用 `Colors.green`。
- 不移除淺色主題。語意色常數為兩主題共用，改值後淺色主題自動跟隨；僅 `_downColorLight` 為淺色主題專屬，維持不變。
